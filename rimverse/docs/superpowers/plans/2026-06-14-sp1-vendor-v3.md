# SP1 — Vendor v3 In, As-Is — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Copy The Dunk Contest v3 wholesale into `~/Sites/thedunkcontest2/dunkcontest/` as its own self-contained app, running byte-identically to production. **No source changes to v3 or to V5/rimverse.**

**Architecture:** Pure relocation + run-wiring. v3 keeps its Express + socket.io + better-sqlite3 stack on `:3000`, isolated from V5's TS workspaces. Verified by boot parity + v3's own test suite + a functional browser gate (SP1 adds no logic, so there are no new unit tests — that would be testing a copy).

**Tech stack:** Node ESM, Express 4, socket.io 4, better-sqlite3 12 (native), `rsync`/`cp`, `git`.

**Spec:** `docs/superpowers/specs/2026-06-14-sp1-vendor-v3-design.md`. **Parent:** `2026-06-14-rimverse-arc-realignment-design.md`.

## Context for the executor

- **Source:** `~/Sites/thedunkcontest/` (v3, deployed). **Destination:** `~/Sites/thedunkcontest2/dunkcontest/` (new folder, **not** a V5 workspace member).
- **Branch:** create `sp1-vendor-v3` off `main` in `~/Sites/thedunkcontest2`. (Do NOT reuse `b2-themed-stages`; that's the dormant B2 PR.)
- **Why a wholesale copy:** v3's static mounts (`/` → `public/`, `/shared` → `shared/`) and importmap resolve relative to `server.js`'s dir; copying the tree intact preserves them. Any reorg risks a client white-screen.
- **The live DB** (`data/dunkcontest.db`, ~16KB real players) is gitignored and not in git — copy it explicitly to preserve players; it stays gitignored.
- **Excludes:** `node_modules` (reinstall — native module), `.git`, v3 dev-meta (`.claude`, `.superpowers`, `docs`), `app.json` (stray Expo stub), and the two dead client files (`public/js/game-client.js`, `public/js/network-manager.js`).
- **Verification commands must surface real exit codes** — do not pipe a gating command (`npm test`, a boot check) through `| tail`/`| head` in an `&&` chain (the pipe masks the exit code). Use `; echo "EXIT=$?"` or check explicitly.

## Task 1: Branch + copy v3 into `dunkcontest/`

**Files:** creates `~/Sites/thedunkcontest2/dunkcontest/` (the vendored tree).

- [ ] **Step 1: Branch off main (from a clean tree)**

Precondition: the realignment planning docs (arc design, SP1 spec, this plan) must already be **committed** — they were authored on disk while the repo was checked out on `b2-themed-stages` (the dormant B2 branch), so confirm the tree is clean before switching or they'll be carried as untracked baggage.

```bash
cd ~/Sites/thedunkcontest2
git status                          # expect clean (planning docs committed on their planning branch / merged to main)
# if stray working changes remain, park them — b2-themed-stages tracked work stays on that branch:
git stash -u 2>/dev/null || true
git checkout main && git pull
git checkout -b sp1-vendor-v3
```

- [ ] **Step 2: Copy v3 wholesale, excluding dev-meta / node_modules / dead code**

```bash
cd ~/Sites/thedunkcontest2
mkdir -p dunkcontest
rsync -a \
  --exclude 'node_modules' --exclude '.git' --exclude '.claude' \
  --exclude '.superpowers' --exclude 'docs' --exclude 'app.json' \
  ~/Sites/thedunkcontest/ dunkcontest/
```
`public/js` is copied **verbatim** — no pruning (SP1's "change nothing" ethos; dead code is inert because nothing imports it). For the record, **7 of the 17 `public/js` files are dead** (unimported by the live graph traced from `main.js`): `game-client.js`, `network-manager.js`, `ball-handler.js`, `dunk-animation-system.js`, `shot-system.js`, `sprite-player.js`, `sprite-player-generator.js`. They ride along harmlessly; cleanup is deferred to a later SP. SP3 must hook the **live** netcode (`net.js`/`game.js`), not these.

- [ ] **Step 3: Verify the tree looks right**

```bash
ls dunkcontest                      # expect: server.js src shared public test tools deploy DEPLOY.md README.md package.json package-lock.json .gitignore data
# public/js is vendored verbatim — confirm the LIVE entry graph is present (main.js is the sole ESM entry):
for f in main net world game hud creator fx stage sprites generator; do test -f dunkcontest/public/js/$f.js && echo "live: $f.js" || echo "MISSING live: $f.js"; done
test -f dunkcontest/data/dunkcontest.db && echo "DB copied" || echo "WARN: no DB (v3 will auto-create empty on first write)"
test -f dunkcontest/.gitignore && grep -qE '^/?data/?$' dunkcontest/.gitignore && echo "nested .gitignore covers data/" || echo "WARN: nested .gitignore missing data/ rule (see Step 4)"
```
Expected: source tree present; all 10 live files present; DB copied; nested `.gitignore` covers `data/`. If the DB is absent (e.g. v3 never ran locally), that's acceptable — the app auto-creates an empty one on first write.

- [ ] **Step 4: Commit the vendored tree (source only; data/ + node_modules excluded by v3's nested .gitignore)**

```bash
cd ~/Sites/thedunkcontest2
git add dunkcontest
git status   # CONFIRM: no dunkcontest/node_modules, no dunkcontest/data/ (incl. -wal/-shm sidecars), no dunkcontest/backups staged
git commit -m "feat(sp1): vendor The Dunk Contest v3 as-is into dunkcontest/ (own app, separate from TS rimverse)"
```
**The load-bearing protection is the nested `dunkcontest/.gitignore`** (copied from v3 — it ignores `node_modules/`, `data/`, `backups/` within the subtree), because that is what covers the SQLite **`-wal`/`-shm` sidecars** that appear the moment the server runs (Task 2/5). The V5 **root** `.gitignore` (`*.db`, un-anchored `node_modules/`) is only a *partial* redundant net — it has no `data/`/`backups/`/`-wal`/`-shm` rules. If `git status` shows any of `dunkcontest/data/`, the WAL sidecars, `backups/`, or `node_modules/` staged, STOP — confirm `dunkcontest/.gitignore` copied and contains those directory rules; as a fallback add **directory** rules `dunkcontest/data/` and `dunkcontest/backups/` (not file-level `*.db`) to the root `.gitignore` so the sidecars + backup tarballs are also caught. Never commit the DB, its sidecars, or node_modules.

## Task 2: Install deps + boot parity

**Files:** creates `dunkcontest/node_modules/` (gitignored).

- [ ] **Step 1: Install v3's deps in the new location (recompiles native better-sqlite3)**

```bash
cd ~/Sites/thedunkcontest2/dunkcontest
node -v   # need >= 18 (native better-sqlite3 ^12 + ESM/importmap); if older, switch Node first
npm ci ; echo "INSTALL_EXIT=$?"
```
Expected: Node ≥18; `INSTALL_EXIT=0`; `better-sqlite3` compiles without error. The lockfile is in sync (lockfileVersion 3), so `npm ci` should succeed; only if it errors on the lockfile, fall back to `npm install`. A native build error means the toolchain (or Node version) is wrong — report it (never copy a prebuilt binary).

- [ ] **Step 2: Boot on a test port and smoke-test the production endpoints**

```bash
cd ~/Sites/thedunkcontest2/dunkcontest
PORT=3999 node server.js & SRV=$!
sleep 2
echo "--- /api/status ---"; curl -sf http://localhost:3999/api/status && echo " OK" || echo " FAIL"
echo "--- / (index) ---";    curl -s -o /dev/null -w '%{http_code}\n' http://localhost:3999/
echo "--- /socket.io client ---"; curl -s -o /dev/null -w '%{http_code}\n' http://localhost:3999/socket.io/socket.io.js
echo "--- /shared/constants.js ---"; curl -s -o /dev/null -w '%{http_code}\n' http://localhost:3999/shared/constants.js
echo "--- /api/leaderboard ---"; curl -sf http://localhost:3999/api/leaderboard >/dev/null && echo "OK" || echo "FAIL"
# clean-exit check: v3 self-exits ≤8s after SIGTERM (server.js:68 flush cap); poll up to 9s for the process to die
kill -TERM $SRV
for i in $(seq 1 9); do kill -0 $SRV 2>/dev/null || break; sleep 1; done
kill -0 $SRV 2>/dev/null && echo "WARN: still running >9s after SIGTERM" || echo "clean exit"
```
Expected: `/api/status` JSON OK; `/` → `200`; `/socket.io/socket.io.js` → `200`; `/shared/constants.js` → `200`; `/api/leaderboard` OK; `clean exit` (an empty-DB shutdown is near-instant; the 8s cap only bites with active sessions). Any `404` on `/shared/...` or the socket.io client means a path-mount regression — re-check the copied layout.

- [ ] **Step 3: Commit (no file changes; this is a verification gate)** — nothing to commit; record the boot results in the task notes.

## Task 4: Isolation from the V5 TS build

- [ ] **Step 1: Confirm `dunkcontest/` is NOT captured by V5's workspaces**

```bash
cd ~/Sites/thedunkcontest2
grep -A3 '"workspaces"' package.json   # expect only ["shared","server","client"] — NOT dunkcontest
npm install --ignore-scripts --dry-run 2>&1 | grep -i dunkcontest && echo "WARN: root install references dunkcontest" || echo "OK: dunkcontest isolated from root workspaces"
```
Expected: `dunkcontest` absent from `workspaces`; root install does not reach into it.

- [ ] **Step 2: Confirm V5 still builds + tests clean (untouched)**

```bash
cd ~/Sites/thedunkcontest2
npm run typecheck ; echo "TYPECHECK_EXIT=$?"
npm test > /tmp/v5-sp1-test.log 2>&1 ; echo "V5_TEST_EXIT=$?"
grep -E "Test Files|Tests " /tmp/v5-sp1-test.log
```
Expected: `TYPECHECK_EXIT=0`; `V5_TEST_EXIT=0`; V5's suite still passes (B2a left it at 216; SP1 changed no V5 source, so unchanged). This proves SP1 didn't disturb the TS app. (Note: capture the exit code BEFORE the `grep` — a `npm test | grep` pipe would mask a failing suite.)

> **Latent isolation note (optional insurance, not required for SP1):** V5's `vitest.config.ts` `include` glob (`*/test/**/*.test.ts`) would structurally match `dunkcontest/test/`, excluded today only because v3's tests are `.test.js` (not `.test.ts`). If you want durable protection against someone later adding a `*.test.ts` under `dunkcontest/`, add `exclude: ['dunkcontest/**']` to `vitest.config.ts`. This is the *one* V5-config touch and is purely defensive — omit it to keep SP1 strictly zero-V5-edit.
