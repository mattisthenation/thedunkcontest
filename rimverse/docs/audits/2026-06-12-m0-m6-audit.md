## H1 — Non-finite input boundary
`Number(msg.mx) || 0` lets `Infinity` (JSON `1e400`) through → `stepPlayer` → `Infinity/Infinity = NaN` → position NaN forever; `aoiPlayers` sort comparator returns NaN → nearest-neighbour ordering corrupted for ALL players; NaN serializes as `null` over the wire. Fix: finite-check at ingestion + defense-in-depth in `clampToArena`/`stepPlayer` + NaN-safe AOI comparator.

## H2 — Intent-flood speed
Tick drains `MAX_INTENTS_PER_TICK=5` intents each with full `dt` → 5× distance. Fix: fold queued movement to the latest vector, integrate **once** per tick; keep action latches separate.

## H3+M1 — Caps
No `MAX_PLAYERS`, no per-socket limit, default `ws` `maxPayload` ~100 MiB. Fix: reject `join` at `MAX_PLAYERS=100`; tight `maxPayload`.

## M2+M3+L3 — Hoop identity
`BallFlight.targetHoop` is a positional index; `reslot()` renumbers on every join/leave; a flight spans many ticks → wrong player penalized, or phantom +2 if the rim vanished. Fix: capture the **defender player id** at launch; penalize that id at landing (void if they left); AOI compares ids.

## M4 — Prediction action-lock
Server integrates movement only when `!p.action` but acks every intent; client predicts walking → 15 Hz snap-back for the whole lock (dunk ≈ 27 ticks). Fix: predictor skips movement/turbo integration while the local anim is an action-lock.
